# 2.7 Event

在 Solidity 中，`event` 用来把合约执行过程中的关键信息写入链上日志（log）。前端、脚本或区块浏览器可以监听这些日志，从而知道合约里发生了什么。

最常见的用法是：函数执行成功后，通过 `emit` 发出一个事件。

## 事件有什么用

- 记录关键操作，例如创建、修改、转账
- 方便前端监听链上变化
- 方便测试和排查交易过程

## 关于 gas 的正确理解

事件并不是“完全不消耗 gas”。emit 事件时依然会消耗 gas，只是事件数据会写入日志（log），而不是像状态变量那样写入合约存储。

这也是为什么事件常用于“通知外部系统”，而不是作为合约内部业务状态的唯一来源。

## 基本语法

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract TodoEventDemo {
    event TodoCreated(uint indexed id, string text);

    string[] public todos;

    function create(string memory text) public {
        todos.push(text);
        emit TodoCreated(todos.length - 1, text);
    }
}
```

## 代码说明

### 1. 声明事件

```solidity
event TodoCreated(uint indexed id, string text);
```

- `TodoCreated` 是事件名
- `id` 和 `text` 是事件参数
- `indexed` 表示这个字段可以被更方便地检索和过滤

### 2. 触发事件

```solidity
emit TodoCreated(todos.length - 1, text);
```

`emit` 的作用就是把这次操作的信息写入日志。这里记录了新待办的编号和内容。

## 什么时候适合用事件

- 合约状态变化后，希望前端立刻感知
- 希望把操作痕迹留在链上日志里
- 希望链下程序按条件筛选某类操作

## 小结

记住两点就够了：

- `event` 是日志机制，不是普通状态变量
- `emit` 事件会消耗 gas，只是写入位置是日志，不是合约存储
