# 2.5 - Enum

在 Solidity 中，`enum`（枚举）用来表示“有限个固定状态”。当一个变量只能在几个选项里取值时，使用 `enum` 会比直接写数字更清楚。

## 1. 基本概念

下面这个例子定义了一个订单状态：

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract EnumExample {
    enum Status {
        Pending,
        Shipped,
        Canceled
    }

    Status public status;

    function ship() public {
        status = Status.Shipped;
    }

    function cancel() public {
        status = Status.Canceled;
    }
}
```

这个 `Status` 枚举只有三种状态：`Pending`、`Shipped`、`Canceled`。

- `Pending`：待处理
- `Shipped`：已发货
- `Canceled`：已取消

也就是说，状态只有三种状态，不存在其他成员。

## 2. 如何使用 enum

在上面的合约里：

- `Status public status;` 声明了一个枚举类型变量
- `ship()` 把状态改成 `Status.Shipped`
- `cancel()` 把状态改成 `Status.Canceled`

如果一个业务场景只有少量固定选项，`enum` 会让代码更容易读懂。

## 3. 默认值

枚举变量如果没有手动赋值，会默认取第一个成员。

在这个例子里，第一个成员是 `Pending`，所以合约部署后 `status` 的初始值就是 `Status.Pending`。

## 4. 小结

- `enum` 用来表示有限个固定状态。
- 这个例子里只有三种状态：`Pending`、`Shipped`、`Canceled`。
- 用 `Status.Shipped` 这样的写法，比直接使用数字更直观。

当你后面学习订单、投票、待办事项等合约时，`enum` 会非常常见。
