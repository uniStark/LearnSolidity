# 2.9 Ethers.js - 安装&实例

`ethers.js` 是一个常用的以太坊 JavaScript 库，可以让我们在本地脚本里连接 RPC、调用合约、发送交易。

这一节用 `Ganache + ethers.js` 演示最基本的本地交互流程。

## 先安装

```bash
npm install ethers
```

并确保本地 Ganache 已经启动，例如 RPC 地址为：

```text
http://127.0.0.1:8545
```

## 交互示例

下面示例演示两件事：

- 写入：调用 `create` 新增一条待办
- 读取：调用 `todoList(0)` 查询第 0 条待办

> `contractAddress` 必须替换成你部署合约后得到的真实地址。
> 这个示例默认你使用支持 ESM 和顶层 `await` 的 Node 运行环境。

```javascript
import { ethers } from "ethers";

const rpcUrl = "http://127.0.0.1:8545";
const provider = new ethers.JsonRpcProvider(rpcUrl);

const abi = [
  "function create(string text)",
  "function todoList(uint256) view returns (string text, bool completed)"
];

const contractAddress = "请替换为你部署后的合约地址";
const signer = await provider.getSigner(0);
const contract = new ethers.Contract(contractAddress, abi, signer);

const tx = await contract.create("Learn Solidity");
await tx.wait();

const todo = await contract.todoList(0);
console.log(todo);
```

## 代码说明

### 1. 连接本地链

```javascript
const provider = new ethers.JsonRpcProvider(rpcUrl);
```

这里使用的是 ethers v6 写法，`JsonRpcProvider` 会连接到 Ganache 提供的本地 RPC。

### 2. 获取签名账户

```javascript
const signer = await provider.getSigner(0);
```

这里取的是 Ganache 第 0 个测试账户，用它来发送交易。

### 3. 使用简洁 ABI

这里没有粘贴一大段 JSON ABI，而是使用人类可读的 ABI 字符串数组。对于入门示例，这样更容易看懂。

### 4. 区分读和写

- `create(...)` 会发送交易，所以需要 `await tx.wait()`
- `todoList(0)` 是只读调用，它使用的是 `Todo[] public todoList;` 自动生成的 getter，不会发起链上交易

## 对应的合约接口

上面的脚本假设你的合约至少有这两个函数：

```solidity
function create(string memory text) public;
function todoList(uint index) public view returns (string memory text, bool completed);
```

## 小结

本地学习时，最常见的组合就是：

- 用 Ganache 提供本地链
- 用 ethers.js 连接 RPC
- 用 signer 发送交易
- 用 Contract 实例读写智能合约
