# 2.6 TodoList - 待办事项

这一节用一个简单的 `TodoList` 合约，把前面学过的 `struct`、数组和函数操作串起来。每条待办事项都包含两部分数据：

- `text`：待办内容
- `completed`：是否完成

这个合约可以看成最基础的 CRUD 示例：

- `create`：新增一条待办
- `update`：修改文字或完成状态
- `del`：做一个重置该位置数据的逻辑删除示例

## 合约示例

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract TodoList {
    struct Todo {
        string text;
        bool completed;
    }

    Todo[] public todoList;

    function create(string memory text) public {
        todoList.push(Todo({text: text, completed: false}));
    }

    function update(uint index, string memory text, bool completed) public {
        require(index < todoList.length, "Invalid index");
        if (bytes(text).length != 0) {
            todoList[index].text = text;
        }
        todoList[index].completed = completed;
    }

    function del(uint index) public {
        require(index < todoList.length, "Invalid index");
        delete todoList[index];
    }
}
```

## 代码说明

### 1. `struct Todo`

`struct` 用来把多个字段组合成一个自定义类型。这里的 `Todo` 表示“一条待办事项”。

### 2. `Todo[] public todoList`

这是一个动态数组，用来保存全部待办事项。加上 `public` 后，Solidity 会自动生成同名 getter，方便按索引查看数据。

### 3. `create`

`create` 会往数组末尾追加一条新数据。新建时默认 `completed` 为 `false`，表示未完成。

### 4. `update`

`update` 先用：

```solidity
require(index < todoList.length, "Invalid index");
```

检查索引是否合法，避免访问不存在的元素。

如果传入的 `text` 不是空字符串，就更新文本；不管文本是否变化，都会更新 `completed` 状态。

这里也有一个取舍：空字符串被当成“不要更新 text”的信号，所以这个设计下，无法主动把文本设置为空字符串。换句话说，传入空字符串时，不会把文本更新成空字符串。

### 5. `del`

`del` 同样先检查索引，再使用 `delete` 重置该位置的数据。这更接近一种逻辑删除，而不是数组里的物理删除。

要注意，`delete` 不会让数组长度变短，而是把该位置重置为默认值：

- `text` 变成空字符串
- `completed` 变成 `false`

## 调用示例

1. 调用 `create("Learn Solidity")`，数组会新增一条待办。
2. 调用 `update(0, "Learn Solidity Basics", true)`，把第 0 条改成已完成。
3. 调用 `del(0)`，第 0 条会被重置为默认值，但数组长度不会变短。

## 小结

这个例子很适合练习三件事：

- 用 `struct` 表达一条完整数据
- 用数组保存多条记录
- 用 `require` 给读写操作加上基本边界检查
