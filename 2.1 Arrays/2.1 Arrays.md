# 2.1 Arrays

在 Solidity 中，数组（`array`）用于按顺序存放一组同类型的数据。初学阶段先掌握四种常见写法：静态数组、动态数组、内存数组和多维数组。

## 1. 静态数组

静态数组在声明时就确定长度，后面不能改大小。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract StaticAndDynamicArrayExample {
    uint[5] public staticArray;
    uint[] public dynamicArray;

    function setStaticValue(uint index, uint value) public {
        staticArray[index] = value;
    }

    function addDynamicValue(uint value) public {
        dynamicArray.push(value);
    }
}
```

这里的 `uint[5] public staticArray;` 表示这个数组只能放 5 个 `uint`，位置从 `0` 到 `4`。

## 2. 动态数组

动态数组长度可以变化，常见操作是 `push()` 和 `pop()`。

上面的例子里：

- `uint[] public dynamicArray;` 是动态数组。
- `push(value)` 会在数组末尾新增一个元素。
- `pop()` 会删除最后一个元素。

动态数组适合元素数量不固定的场景。

要注意：数组索引无效时会 `revert`，不会返回空值或默认值；这个规则对静态数组、动态数组和多维数组都一样。

## 3. 内存数组

有些数组只想在函数执行时临时使用，这时可以创建 `memory` 数组。它不会永久写入链上状态。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract MemoryArrayExample {
    function createMemoryArray(uint size) public pure returns (uint[] memory) {
        uint[] memory memoryArray = new uint[](size);
        for (uint i = 0; i < size; i++) {
            memoryArray[i] = i;
        }
        return memoryArray;
    }
}
```

这段代码会创建一个长度为 `size` 的临时数组，然后把每个位置填成自己的下标值。

## 4. 多维数组

多维数组就是“数组里面再放数组”。最常见的是二维数组，可以把它理解成表格。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract MultiArrayExample {
    uint[][] public table;

    function addRow(uint[] memory row) public {
        table.push(row);
    }

    function getValue(uint row, uint col) public view returns (uint) {
        return table[row][col];
    }
}
```

`table[row][col]` 先选第几行，再选这一行里的第几个元素。

## 5. 小结

- 静态数组：长度固定。
- 动态数组：长度可变。
- 内存数组：只在函数执行期间存在。
- 多维数组：数组中再包含数组。

数组是后续学习 `mapping`、`struct` 和更复杂数据结构的基础，先把“长度是否固定”和“数据存在哪里”这两个点分清楚最重要。
