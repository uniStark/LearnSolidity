# 1.6 Constants

在 Solidity 中，`constant` 表示常量。常量的值在声明时就要确定，后续不能再修改。

使用常量有两个直接好处：

1. 代码更容易读，一眼就能看出某个固定值的含义。
2. 不需要写入合约存储，通常比普通状态变量更省 Gas。

## 如何定义常量

常量使用 `constant` 关键字定义，并且必须在声明时赋值。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract Constants {
    uint256 public constant MY_UINT = 100;
    address public constant MY_ADDRESS = 0x1234567890123456789012345678901234567890;
    bytes32 public constant MY_BYTES32 =
        0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef;
}
```

这个例子里演示了三种常见常量：

- `uint256`：保存整数常量。
- `address`：保存固定地址。
- `bytes32`：保存固定长度的 32 字节数据。

## 小结

`constant` 适合那些在编译期就已经确定的固定值。相比之下，`immutable` 用于部署时才能确定、但部署后不再变化的值。
