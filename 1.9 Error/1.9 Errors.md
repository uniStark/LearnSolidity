# 1.9 Errors

在 Solidity 中，常见的报错方式有 `revert`、`require`、`assert`，以及自定义错误（custom errors）。

它们都可以让交易失败并回退状态，但用途并不完全一样：

- `revert`：适合手动终止执行
- `require`：适合检查输入和前置条件
- `assert`：适合检查“理论上永远应该成立”的内部条件
- 自定义错误：更节省 Gas，也能携带参数

需要注意的是：自定义错误从 Solidity `0.8.4` 开始支持。

## 示例合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract Errors {
    error MyError(string message, uint256 input);

    function testRevert(uint256 value) public pure returns (uint256) {
        if (value == 0) {
            revert("input can not equal 0");
        }
        return value;
    }

    function testRequire(uint256 value) public pure returns (uint256) {
        require(value != 0, "input can not equal 0");
        return value;
    }

    function testAssert() public pure returns (uint256) {
        uint256 a = 2;
        uint256 b = 3;
        uint256 sum = a + b;

        assert(sum == 5);
        return sum;
    }

    function testCustomError(uint256 value) public pure returns (uint256) {
        if (value == 0) {
            revert MyError("input can not equal 0", value);
        }
        return value;
    }
}
```

## 代码说明

`testRevert()` 里使用 `revert("...")` 主动回退，并返回错误信息。

`testRequire()` 里使用 `require(...)` 检查输入是否有效。这是实际开发中最常见的写法之一。

`testAssert()` 里使用 `assert(...)` 检查内部不变量。这里断言 `2 + 3` 的结果必须等于 `5`，这类检查是写给开发者自己的，不应该拿来做普通用户输入校验。

`testCustomError()` 里使用了自定义错误 `MyError`。和长字符串相比，这种写法通常比长字符串错误信息更省 Gas，而且还能把 `value` 一起带出去，方便排查问题。

## 小结

写合约时，优先把输入检查写清楚。一般来说：

- 参数校验常用 `require`
- 需要主动中断时可以用 `revert`
- 检查内部逻辑错误时才用 `assert`
- 想进一步优化错误信息成本时，可以使用自定义错误
