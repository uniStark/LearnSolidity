# 2.3 Mapping - 映射

在 Solidity 中，`mapping` 可以理解成“键值对表”。你提供一个键（key），合约就能找到对应的值（value）。它很像字典或哈希表，常用于余额、权限和地址到数据的映射。

## 1. 基本概念

`mapping` 的语法如下：

```solidity
mapping(keyType => valueType) variableName;
```

例如：

```solidity
mapping(address => uint) public balances;
```

这表示把一个地址映射到一个 `uint` 数值，常见写法就是记录某个地址对应的余额。

## 2. 示例合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.7;

contract MappingExample {
    mapping(address => uint) public balances;

    function setBalance(address account, uint balance) public {
        balances[account] = balance;
    }

    function deleteBalance(address account) public {
        delete balances[account];
    }
}
```

这个合约演示了两件事：

- 通过 `balances[account] = balance;` 设置某个地址的值。
- 通过 `delete balances[account];` 把这个地址对应的值重置为默认值。

这里的 `setBalance()` 只演示语法，不代表真实项目里的权限设计。

## 3. 默认值

`mapping` 里的键不需要提前创建。即使你从来没有给某个键赋值，读取它时也会得到该类型的默认值。

- `uint` 的默认值是 `0`
- `bool` 的默认值是 `false`
- `address` 的默认值是 `0x0000000000000000000000000000000000000000`

所以如果某个地址没有设置过余额，`balances[address]` 读出来会是 `0`。

最大的初学者误区是：读到 `0` 时，不能判断这个 key 是从未赋值过，还是被显式写成了 `0`。

## 4. 删除

`mapping` 不能像数组那样真正“移除一个键”，但可以把这个键对应的值恢复为默认值：

```solidity
delete balances[account];
```

对于 `mapping(address => uint)` 来说，删除后的结果就是这个地址对应的值重新变成 `0`。

## 5. 为什么不能遍历

`mapping` 只能通过键去查值，不能直接知道“里面一共有多少项”，也不能直接 `for` 循环遍历所有键。

这也是初学者最容易忽略的一点：

- 你可以查 `balances[msg.sender]`
- 但你不能直接列出所有出现过的地址

如果业务需要遍历，通常要额外配合一个数组来单独保存键。

## 6. 存储位置

mapping 不能在 memory 中创建，通常作为状态变量使用，也可以作为 storage 中结构体的成员。

## 7. 小结

- `mapping` 用来保存键值对。
- 没有赋值的键，读取时会返回默认值。
- `delete` 会把值重置为默认值。
- `mapping` 不能直接遍历。
- `mapping` 存在于 `storage` 中。
